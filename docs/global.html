checkPermission(permission, req, res, next) → {Promise.<void>}

Middleware para verificar si un usuario tiene un permiso específico.
Parameters:
Name Type Description
permission string El permiso que se desea verificar.
req Object El objeto de solicitud.
res Object El objeto de respuesta.
next function La función del siguiente middleware.
Source:
  • middlewares/authMiddleware/checkPermission.js, line 35
Returns:
- Resuelve si el usuario tiene el permiso, de lo contrario envía una respuesta 403.
Type
Promise.<void>
Example
// Supongamos que tenemos un servidor Express configurado y una ruta protegida que requiere el permiso 'admin'.
app.get('/ruta-protegida', verifyToken, checkPermission('admin'), (req, res) => {
  res.status(200).json({ message: 'Acceso concedido' });
});

// Luego, desde el cliente, podríamos hacer una solicitud GET a esta ruta con el token de acceso en el encabezado:
const response = await fetch('/ruta-protegida', {
  method: 'GET',
  headers: {
    'Authorization': 'Bearer ' + accessToken
  }
});

// Si el usuario tiene el permiso 'admin', la respuesta tendrá un estado 200 y un mensaje de éxito.
if (response.status === 200) {
  const data = await response.json();
  console.log('Acceso concedido:', data.message);
} else {
  const error = await response.json();
  console.error('Error de acceso:', error.message);
}

Inicia sesión de un usuario con el correo electrónico y la contraseña proporcionados.

Name Type Description

Properties
Name Type Description

Properties
Name Type Description
email string El correo electrónico del usuario que intenta iniciar sesión.
password string La contraseña del usuario que intenta iniciar sesión.
res Object El objeto de respuesta utilizado para enviar el resultado del inicio de sesión.
Source:
  • controllers/authController/login.js, line 41
Returns:
- Una promesa que se resuelve cuando el inicio de sesión es exitoso o se rechaza con un mensaje de error.
Type
Promise.<void>
Example
// Supongamos que tenemos un servidor Express configurado y una ruta para el inicio de sesión.
app.post('/login', login);

// Luego, desde el cliente, podríamos hacer una solicitud POST a esta ruta con el siguiente cuerpo:
const response = await fetch('/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'usuario@example.com',
    password: 'contraseña123'
  })
});

// Si el inicio de sesión es exitoso, la respuesta tendrá un estado 200 y contendrá el token de acceso.
if (response.status === 200) {
  const data = await response.json();
  console.log('Token de acceso:', data.accessToken);
} else {
  const error = await response.json();
  console.error('Error de inicio de sesión:', error.message);
}
